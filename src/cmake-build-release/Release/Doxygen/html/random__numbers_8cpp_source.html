32 
33  CoastalME's two random number generators are maximally equidistributed combined Tausworthe generators. The code is modified from taus.c in gsl-1.9, the GNU Scientific Library. The sequence is
34 
35  x_n = (s1_n ^ s2_n ^ s3_n)
36 
37  s1_{n+1} = (((s1_n & 4294967294) <<12) ^ (((s1_n <<13) ^ s1_n) >> 19))
38  s2_{n+1} = (((s2_n & 4294967288) << 4) ^ (((s2_n << 2) ^ s2_n) >> 25))
39  s3_{n+1} = (((s3_n & 4294967280) <<17) ^ (((s3_n << 3) ^ s3_n) >> 11))
40 
41  computed modulo 2^32. In the three formulae above '^' means exclusive-or (C-notation), not exponentiation. Note that the algorithm relies on the properties of 32-bit integers (it is formally defined on bit-vectors of length 32). GNU have added a bitmask to make it also work on 64 bit machines.
42 
43  Each generator is initialized with
44 
45  s1_1 .. s3_1 = s_n MOD m
46 
47  where s_n = (69069 * s_{n-1}) mod 2^32, and s_0 = s is the user-supplied seed.
48 
49  The theoretical value of x_{10007} is 2733957125. The subscript 10007 means (1) seed the generator with s=1 (2) do six warm-up timesteps, (3) then do 10000 actual timesteps. The period of this generator is about 2^88 i.e. 309485009821345068724781056
50 
51  From: P. L'Ecuyer, "Maximally equidistributed combined Tausworthe generators", Mathematics of Computation 65, 213 (1996), 203-213. This is available on the net from L'Ecuyer's home page:
52 
53  http://www.iro.umontreal.ca/~lecuyer/myftp/papers/tausme.ps
54  ftp://ftp.iro.umontreal.ca/pub/simulation/lecuyer/papers/tausme.ps
55 
56  Update: April 2002
57 
58  There is an erratum in the paper "Tables of Maximally Equidistributed Combined LFSR Generators", Mathematics of Computation, 68, 225 (1999), 261-269. See http://www.iro.umontreal.ca/~lecuyer/myftp/papers/tausme2.ps
59 
60  "... the k_j most significant bits of z_j must be non-zero, for each j. (Note: this restriction also applies to the computer code given in [4], but was mistakenly not mentioned in that paper.)"
61 
62  This affects the seeding procedure by imposing the requirement s1 > 1, s2 > 7, s3 > 15.
63 
64  Update: November 2002
65 
66  There was a bug in the correction to the seeding procedure for s2. It affected the following seeds 254679140 1264751179 1519430319 2274823218 2529502358 3284895257 3539574397 (s2 < 8).
67 
68 ===============================================================================================================================*/
69 unsigned long CSimulation::ulGetRand0(void)
70 {
71  // ulGetRand0() generates random numbers
72  m_ulRState[0].s1 = ulGetTausworthe(m_ulRState[0].s1, 13, 19, 4294967294ul, 12);
73  m_ulRState[0].s2 = ulGetTausworthe(m_ulRState[0].s2, 2, 25, 4294967288ul, 4);
74  m_ulRState[0].s3 = ulGetTausworthe(m_ulRState[0].s3, 3, 11, 4294967280ul, 17);
75 
76  return (m_ulRState[0].s1 ^ m_ulRState[0].s2 ^ m_ulRState[0].s3);
77 }

97 {

100 
101  m_ulRState[0].s1 = ulGetLCG(ulSeed);
102  if (m_ulRState[0].s1 < 2)

104

106  if (m_ulRState[0].s2 < 8)
107  m_ulRState[0].s2 += 8UL;
108

110  if (m_ulRState[0].s3 < 8)
111  m_ulRState[0].s3 += 16UL;
112 
113  // Warm it up

116

157 /*===============================================================================================================================
158 
159  Using ulGetRand0(), randomly samples from a unit Gaussian (normal) distribution, is routine gasdev.c in:
160 
161  Press, W.H., Teukolsky, S.A., Vetterling, W.T. and Flannery, B.P. (1992). Numerical Recipes in C (Second Edition), Cambridge University Press, Cambridge. 994 pp.
162 
163 ===============================================================================================================================*/
164 double CSimulation::dGetRand0Gaussian(void)
165 {
166  static int snSet = 0;
167  static double sdGset;
168  double dRet;
169 
170  if (0 == snSet) // we don't have an extra deviate handy, so
171  {
172  double dFac, dRsq, dV1, dV2;
173 
174  do
175  {
176  // Pick two uniform numbers in the square extending from -1 to +1 in each direction, see if they are in the unit circle
177  dV1 = 2 * dGetRand0d1()-1; // uses ulGetRand0()
178  dV2 = 2 * dGetRand0d1()-1; // ditto
179  dRsq = dV1 * dV1 + dV2 * dV2;
180  }
181  while (dRsq >= 1 || 0 == dRsq); // if they are not, try again
182 
183  dFac = sqrt(-2 * log(dRsq)/dRsq);
184 
185  // Now make the Box-Muller transformation to get two normal deviates, return one and save the other for next time
186  sdGset = dV1 * dFac;
187  snSet = 1; // set flag
188  dRet = dV2 * dFac;
189  }
190  else
191  {
192  snSet = 0; // we have an extra deviate handy so unset the flag and return it
193  dRet = sdGset;
194  }
195 
196  return (dRet);
197 }

207 void CSimulation::Rand1Shuffle(int* nArray, int nLen)
208 {
209  nLen--;
210  while (nLen > 0)
211  {
212  int n1 = nGetRand1To(nLen); // uses Rand1()
213  int nTmp = nArray[n1];
214  nArray[n1] = nArray[nLen];
215  nArray[nLen--] = nTmp;
216  }
217 }
